What it covers

A skill shaped like a real Android CLI session.

The skill is intentionally narrow: it covers the repetitive Android command-line work that benefits from strong defaults, progressive disclosure, and a few non-obvious gotchas.

01

Find the correct root

Starts cheap, stops early, and avoids broad scans until they are justified.

02

Validate with the wrapper

Targets `./gradlew` help, build, lint, and test commands instead of wandering through tasks menus.

03

Control devices sanely

Uses explicit serial targeting, bounded log capture, and finite reproduction loops.

04

Triage UI with less context

Defaults to reduced screenshots, only opens XML when needed, and treats full dumps as a last resort.

05

Modernize without cargo cults

Frames upgrades as a compatibility problem across Gradle, AGP, Kotlin, JDK, and compileSdk.

06

Keep the top-level file lean

The dispatcher stays short. Heavier details live in focused reference files loaded only when relevant.

Install and run

Three commands, one entry point.

The skill ships as a normal installable package from this repository. It works best when the agent needs Android-specific context it would not reliably infer on its own.
